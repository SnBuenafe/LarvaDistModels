We used Climate Data Operators (cdo) to process the netcdf files from ESGF (Earth System Grid Federation) MetaGrid. To do this in R, we used hotrstuff, an R package that we are currently developing to more easily wrangle Earth System Model (ESM) data in R.

To rerun all of this, the user must: - [x] define the file paths. Here we purposely used two different file paths to spread the memory load because some models take up to 700gb of storage space. - [x] define the variable that will be used and use the proper ESGF code (e.g., “tos”, “o2os”, “thetao”, etc.)

Code 
library(hotrstuff)
library(terra)

# Properly define file paths as necessary.
# variable = "tos" # Change this as needed

1. Download model outputs

wget scripts are downloaded from ESGF (Earth System Grid Federation) MetaGrid. For this analyses we used Ocean Model Intercomparison Project Phase 2 (OMIP2).

2. Shift years

OMIP runs are six cycles of simulations using the 61-year JRA55-do forcing data set (1958-2018), resulting in 366-year model outputs. So, we need to shift the years 1652 years forward for some of the models (e.g., starting at 0001) to standardize across models (see Tsujino et al., 2020). Note: When working with depth-resolved models, I would highly suggest running steps 2-3 per model unless you’re working on an HPC or a very powerful laptop/workstation.

Code 
htr_shift_years(indir = file.path(volumes_dir, "raw", "omip", "vo_temporary"),
                outdir = file.path(volumes_dir, "proc", "shifted", "omip", variable),
                adjust_value = 1652)

3. Merge

Now, we merge the files by model-variable-scenario-frequency combination and only take the files that have the years 1956-1981 to match the Nishikawa dataset.

Code 
htr_merge_files(indir = file.path(volumes_dir, "proc", "shifted", "omip", variable), #input directory
                outdir = file.path(base_dir, "data", "proc", "merged", "omip", variable), #output directory
                year_start = 1956, # earliest year considered
                year_end = 1981 # latest year considered
            )

4. Select years

Because we’re using OMIP models with six simulations of 1958-2018, we need to account for the spin-up of the models and the fact that after each cycle, the model experiences a large shock from 2018 going back to 1958 (see Huguenin et al., 2022). So, even though the Nishikawa dataset is from 1956-1981, we are going to calculate predictors from 1963 (+5 years from the initial year of each simulation cycle) until 1981, resulting in a 19-year summary.

Code 
htr_slice_period(indir = file.path(base_dir, "data", "proc", "merged", "omip", variable), #input directory
                 outdir = file.path(base_dir, "data", "proc", "sliced", "omip", variable), #output directory
                 freq = "Omon", #ocean, monthly
                 scenario = "omip2",
                 year_start = 1963,
                 year_end = 1981,
                 overwrite = FALSE
             )

5. Fix calendar periods (if needed)

This function fixes the calendar days, for days with leap years. This is to standardize the calendar across the different models.

## 2. Shift years

OMIP runs are six cycles of simulations using the 61-year JRA55-do forcing data set (1958-2018), resulting in *366-year* model outputs. So, we need to shift the years 1652 years forward for some of the models (e.g., starting at 0001) to standardize across models (see [Tsujino et al., 2020](https://gmd.copernicus.org/articles/13/3643/2020/)). *Note*: When working with depth-resolved models, I would highly suggest running steps 2-3 per model unless you're working on an HPC or a very powerful laptop/workstation.

## 4. Select years
